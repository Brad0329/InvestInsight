# 테마 투자 분석 웹앱 — 개발 컨텍스트

> VS Code + Claude 작업 시작 전 반드시 읽을 것.
> 이 문서는 사전 리서치 및 기획 단계에서 정리된 내용으로, 개발 방향의 기준이 됩니다.

---

## 1. 프로젝트 개요

### 목적
사용자가 직접 투자 테마를 정의하고, 테마 내 종목들의 밸류체인 관계를 시각화하며,
DART 공식 공시 데이터·뉴스·IR 자료를 조회하고, AI와 대화하며 투자 분석을 수행하는
**범용 테마 투자 분석 웹앱**.

수소, 2차전지, 반도체, AI인프라, 방산 등 **어떤 테마도 담을 수 있는 구조**가 핵심.

### 타겟 사용자
- 특정 산업 테마에 집중 투자하는 개인 투자자
- 주식 투자 경험은 있으나 개별 산업의 기술/밸류체인 이해가 부족한 투자자
- 여러 종목을 비교하며 중장기 투자 관점에서 분석하고 싶은 투자자

### 핵심 가치
- 분기보고서 PDF를 직접 읽지 않아도 핵심 재무 데이터를 즉시 확인
- AI와 대화하며 재무 데이터에 대한 질문/분석 가능
- 테마 내 종목들의 밸류체인 포지션을 한눈에 파악
- 뉴스·IR 자료를 함께 확인하며 정성적 판단 보완

## 3. 화면 구조 (3단 레이아웃)

웹 기반, 대형 모니터/태블릿 최적화. 모바일은 핵심 요약 + 채팅만 보여주는 반응형.

```
┌─────────────┬──────────────────────────────┬─────────────────┐
│  LEFT PANEL │        CENTER PANEL          │  RIGHT PANEL    │
│  (240px)    │        (유동적)               │  (360px)        │
│             │                              │                 │
│  테마 목록   │  [탭] 밸류체인|재무|뉴스|IR  │  AI 채팅        │
│  ──────────  │                              │                 │
│  > 수소      │  밸류체인 뷰:                │  [모델 선택]    │
│  > 2차전지   │  생산→저장→활용 흐름도       │  Claude / GPT   │
│  > 반도체    │  + 각 단계별 종목 배치       │  DeepSeek 등    │

│  > 방산      │  재무 뷰:                    │  [질문 입력창]  │

│             │  + AI 자동 핵심요약 카드      │                 │

│  LS머트리얼즈│                              │                 │
│             │  IR 뷰:                      │                 │
│             │  DART 기업설명회 자료         │                 │
│             │  + IR 홈페이지 링크           │                 │
└─────────────┴──────────────────────────────┴─────────────────┘
```

### 사용자 플로우
1. 좌측에서 테마 선택 → 밸류체인 시각화 + 종목 목록 표시
2. 종목 클릭 → 공시 목록 표시 (분기/반기/사업보고서)
3. 분기보고서 선택 → 재무 탭에 데이터 표시 + AI 핵심요약 자동 생성
4. 뉴스 탭: 네이버 뉴스 + DART 최신 공시 (서브탭으로 구분)
5. IR 탭: DART 기업설명회 자료(있는 경우만 표시) + IR 홈페이지 링크
6. 우측 채팅창에서 AI에게 질문 → 재무 데이터 컨텍스트 기반 답변

### 페이지 구성
| 경로 | 설명 |
|------|------|
| `/` | 메인 분석 화면 (3단 레이아웃) |
| `/admin/themes` | 테마 및 종목 관리 페이지 |
| `/settings` | API 키 설정, AI 모델 선택 |

## 4. DART OpenAPI 연동 스펙

**기본 정보**
- 베이스 URL: `https://opendart.fss.or.kr/api/`
- 키 발급: https://opendart.fss.or.kr (무료, 즉시 발급)
- 일 호출 한도: 10,000건
- **주의**: 브라우저 직접 호출 불가 (CORS). Vercel Edge Function으로 프록시 처리 필요 → 개발 시작 전 최우선 해결.

### 4-1. 회사 검색
```
GET /company.json?corp_name=한선엔지니어링&api_key={KEY}
```

### 4-2. 공시 목록 조회
```
GET /list.json
  ?corp_code=00958451
  &pblntf_ty=A    ← 정기공시 (분기/반기/사업보고서)
  &pblntf_ty=F    ← 기타공시 (기업설명회 IR 자료 포함)
  &api_key={KEY}

주요 응답 필드:
  rcpNo     - 접수번호
  reportNm  - "분기보고서", "기업설명회자료" 등
  rcptDt    - 접수일자
```

### 4-3. 재무제표 조회
```
GET /fnlttSinglAcntAll.json?rcpNo=...&repTp=11011&api_key={KEY}

repTp 코드:
  11011 - 재무상태표
  11012 - 손익계산서
  11013 - 포괄손익계산서
  11014 - 자본변동표
  11015 - 현금흐름표
```

## 5. XBRL 파싱

DART는 각 보고서에 대해 XBRL 파일 패키지도 제공. PDF보다 구조화되어 항목별 수치 직접 추출 가능. AI 핵심요약 자동 생성 시 이 데이터를 컨텍스트로 전달.

### 주요 XBRL 항목명 (ifrs-full 표준)
| 항목명 | 설명 |
|--------|------|
| `Assets` | 자산총계 |
| `CurrentAssets` | 유동자산 |
| `Liabilities` | 부채총계 |
| `Equity` | 자본총계 |
| `CashAndCashEquivalents` | 현금및현금성자산 |
| `PropertyPlantAndEquipment` | 유형자산 |
| `Revenue` | 매출액 |
| `GrossProfit` | 매출총이익 |
| `OperatingIncomeLoss` | 영업이익 |
| `ProfitLoss` | 당기순이익 |

### 파싱 핵심 코드 (Python)
```python
import xml.etree.ElementTree as ET

tree = ET.parse('entity00958451_2025-09-30.xbrl')
root = tree.getroot()

for elem in root.iter():
    ctx = elem.get('contextRef', '')
    val = elem.text
    if val and val.strip().lstrip('-').isdigit():
        tag = elem.tag.split('}')[-1]
        print(f"{tag}: {int(val)}")
```

## 6. AI 연동 스펙

### 멀티 AI 지원 구조
금융 특화 모델(BloombergGPT 등)은 외부 API 미제공 또는 영미권 데이터 중심이라 실용성 낮음. 한국어 공시 분석에는 범용 최신 모델이 더 적합.

| AI | 모델 | 특징 |
|----|------|------|
| Claude (Anthropic) | claude-sonnet-4-6 | 긴 컨텍스트, 문서 분석 강점 |
| GPT-4o (OpenAI) | gpt-4o | 범용, 생태계 가장 큼 |
| DeepSeek | deepseek-reasoner | 추론 강점, 저렴, OpenAI API 호환 |
| Gemini (Google) | gemini-2.0-flash | 멀티모달, 긴 컨텍스트 |

DeepSeek은 OpenAI SDK와 호환되어 베이스 URL만 변경하면 동작.

### API 키 관리
사용자가 `/settings` 페이지에서 본인 API 키 직접 입력 → localStorage 저장. 비용 각자 부담. 로그인 불필요.

### AI 호출 구조 (Claude 예시)
```javascript
const response = await fetch("https://api.anthropic.com/v1/messages", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    model: "claude-sonnet-4-6",
    max_tokens: 1000,
    system: SYSTEM_PROMPT,
    messages: conversationHistory  // 대화 히스토리 전체 포함
  })
});
```

### 시스템 프롬프트 구조
```
당신은 테마 투자 분석 전문가입니다.
아래 재무 데이터를 바탕으로 투자자의 질문에 답하세요.

[회사 정보]
- 회사명: {company_name} / 종목코드: {stock_code}
- 보고서: {report_period}

[재무 데이터]
{financial_data_json}

규칙:
- 특정 종목 매수/매도 추천은 하지 않습니다
- 객관적 팩트 기반으로 분석합니다
- 전문 용어는 괄호 안에 쉬운 설명을 추가합니다
```

### AI 핵심요약 자동 생성
분기보고서 선택 시 XBRL 수치 데이터를 AI에 전달 → 투자자 관점 핵심 포인트 카드 자동 생성.
(DART 원문에는 이런 요약이 없음. AI가 수치를 분석해 직접 생성하는 부가 기능.)

예시 출력:
- 📈 영업이익률 급개선 — 10.8% → 15.5% (YoY +4.7%p)
- 🏗️ 신공장 건설 진행중 — 건설중인자산 282억, 2025.11 완공 예정
- ⚠️ 전환사채 리스크 — CB 210억 발행, 부채비율 34%→63%

## 8. IR 자료 연동 스펙

### IR 자료 성격
- **의무 공시 아님** → 대형주는 올리는 경우 많으나, 중소형·코스닥 소형주는 없는 경우가 대부분
- 한선엔지니어링 같은 최근 상장 소형주는 DART에 IR 자료 없을 가능성 높음

### 처리 방식
- DART `pblntf_ty=F` (기타공시)로 기업설명회 자료 조회 → **있으면 표시, 없으면 섹션 자체 숨김**
- 각 종목의 IR 홈페이지 URL은 테마 관리 페이지에서 수동 입력 → 외부 링크로 연결

## 9. 테마/종목 관리

### 테마 분류 체계 결정사항
- KRX 공식 업종 코드: 너무 거시적 → 투자 분석용 부적합
- 한투 KIS API 테마코드: **2021년 이후 관리 중단** → 사용 불가
- **키움 OpenAPI+**: 현재 유일하게 살아있는 테마 데이터
  - `GetThemeGroupList()` — 전체 테마 목록 + ID 반환
  - `GetThemeGroupCode('ID')` — 특정 테마 소속 종목 코드 리스트 반환
  - 단, Windows + OCX 방식 → 웹앱 직접 호출 불가
  - **활용**: Python 스크립트로 1회 추출 → JSON 저장 → 앱 초기 마스터 데이터
- **결론**: 키움으로 초기 데이터 추출 + `/admin/themes` 페이지에서 이후 수동 관리

### JSON 데이터 구조
```json
{
  "themes": [
    {
      "id": "hydrogen",
      "name": "수소",
      "description": "수소 생산·저장·운송·활용 밸류체인 전반",
      "stocks": [
        {
          "name": "한선엔지니어링",
          "code": "452280",
          "corp_code": "00958451",
          "value_chain": "연료전지 부품 (SOFC 배관 모듈)",
          "ir_url": "https://회사홈페이지/ir",
```

### 테마 관리 페이지 기능 (`/admin/themes`)
- 테마 추가 / 수정 / 삭제
- 종목 추가: 종목코드 입력 → DART API로 회사명 자동 조회
- 종목별 밸류체인 포지션, IR 홈페이지 URL, 메모 입력
- 종목 삭제
- JSON export (백업용)

> **향후 확장**: AI로 사업보고서 자동 분석 → 테마 자동 분류 제안 기능 추가 가능

## 10. 밸류체인 시각화

### 방향 결정
- NotebookLM 등 외부 AI 도구는 밸류체인 그림 생성 불가 (문서 질의응답 도구)
- 웹앱 내에서 직접 구현하는 방식 채택
- **초기: Mermaid.js → 추후: Reactflow로 업그레이드**

### 단계별 구현 계획

**1단계 (초기): Mermaid.js**
텍스트로 다이어그램 정의 → 자동 렌더링. 구현 가장 간단.

```javascript
// 수소 밸류체인 예시
graph LR
  A[생산\n그린/블루수소] --> B[저장/운송\n압축/액화/암모니아]
  B --> C[충전 인프라]
  B --> D[연료전지\nSOFC/PEMFC]
  C --> E[모빌리티\n수소차/버스]
  D --> F[발전/산업용]
```

각 노드에 종목 태그 연결 → 클릭 시 해당 종목 상세로 이동.

**2단계 (업그레이드): Reactflow**
노드-엣지 기반 인터랙티브 플로우 차트. 드래그, 줌, 커스텀 노드 등 지원.
종목 카드를 노드로 직접 표현 가능.

### 테마별 밸류체인 데이터 구조
각 테마 JSON에 밸류체인 단계 정의 추가:

```json
    {
      "name": "한선엔지니어링",
      "code": "452280",
      "stage_id": "fuel_cell"
    }
  ]
}
```

## 12. AI 채팅 질문 템플릿

채팅창 위에 카테고리 버튼 → 클릭 시 하위 질문 목록 표시 → 선택 시 입력창 자동 입력되는 구조.
총 8개 대분류, 40개 질문.

### 1. 재무 건전성
> "이 회사 재무가 튼튼한가?"

- 최근 3분기 영업이익률 추이와 개선/악화 원인은?
- 부채비율 변화와 현재 수준이 업종 평균 대비 위험한가?
- 영업이익은 흑자인데 현금흐름이 마이너스인 이유는?
- 일회성 손익(파생상품, 자산매각 등) 제거 시 실질 수익성은?
- 이자보상배율(영업이익/이자비용)로 본 부채 상환 능력은?

### 2. 성장성
> "이 회사는 앞으로 커질 수 있는가?"

- 매출 성장이 정체되거나 둔화된 이유는?
- 신사업/신제품의 현재 매출 기여도와 본격 성장 예상 시점은?
- 수주잔고 또는 파이프라인 규모와 향후 매출 전환 가능성은?
- 고객사가 성장하면 이 회사 매출에 어떤 영향을 주는가?
- 해외 매출 비중과 확대 가능성은?

### 3. 사업 경쟁력
> "이 회사는 대체될 수 없는가?"

- 밸류체인 내 이 회사의 포지션과 교체 가능성(대체 리스크)은?
- 핵심 고객사 매출 의존도가 지나치게 높지 않은가?
- 경쟁사 대비 기술/특허/인증 측면의 차별점은?
- 주요 고객사가 이탈하거나 내재화할 가능성은?
- 원가 경쟁력 — 원재료 가격 변동에 얼마나 노출되어 있는가?

### 4. 리스크
> "어떤 상황이 오면 이 회사가 위험해지는가?"

- 이 회사의 최대 리스크 3가지를 꼽는다면?
- 전환사채/유상증자로 인한 주식 희석 가능성과 규모는?
- 업황이 급격히 나빠지는 시나리오와 이 회사의 대응력은?
- 대주주 지분 변동이나 경영권 이슈가 있는가?
- 규제 변화(정부 정책 축소 등)가 사업에 미치는 영향은?

### 5. 투자 타이밍
> "지금 사도 되는가, 기다려야 하는가?"

- 현재 PER/PBR이 과거 밴드 대비 저평가/고평가 어느 쪽인가?
- 주가 상승의 촉매(카탈리스트)가 될 수 있는 이벤트는?
- 최근 주가 급락 시기와 원인 — 그때와 지금 상황이 다른가?
- 실적 발표, 수주 공시 등 주요 일정이 언제인가?
- 지금 진입 시 감당해야 할 하방 리스크는 어느 정도인가?

### 6. 테마/밸류체인
> "이 테마에서 어떤 종목이 가장 유리한가?"

- 테마 내에서 이 회사가 가장 직접적으로 수혜받는 구간은?
- 테마 성장의 과실이 가장 크게 돌아오는 밸류체인 단계는?
- 테마 자체가 꺾일 수 있는 시나리오와 신호는?
- 정부 정책/보조금 의존도가 높아 지속성이 불확실하지 않은가?
- 글로벌 동종 업체들의 밸류에이션과 비교하면?

### 7. 종목 비교
> "같은 테마 안에서 어디에 투자하는 게 나은가?"

- A사와 B사 중 영업이익률이 더 높은 곳은?
- 같은 테마 내 부채비율이 가장 낮은 종목은?
- 시가총액 대비 매출액(PSR) 기준 가장 저평가된 종목은?
- 성장성과 안정성을 동시에 고려하면 어떤 종목이 우선인가?
- 단기 vs 장기 보유 관점에서 각각 유리한 종목은?

### 8. 종합 판단
> "결론적으로 어떻게 볼 것인가?"

- 이 회사의 투자 thesis(핵심 투자 논거)를 한 문단으로 요약하면?
- 지금 시점에서 매수/관망/제외 중 어느 판단이 합리적인가?
- 이 투자가 틀렸다는 신호(청산 기준)는 무엇인가?
- 6개월~1년 내 주목해야 할 체크포인트는?
- 포트폴리오에서 이 종목의 적정 비중은?
